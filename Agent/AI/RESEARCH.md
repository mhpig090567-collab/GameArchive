### 2. 레퍼런스 플랫폼 분석

실제 운영 중인 웹 게임 플랫폼 3곳의 메인 화면 구성을 비교 분석했다.

| 구분 | CrazyGames (웹 게임) | itch.io (인디 게임) | Steam (PC 게임 스토어) |
| ------ | --------------------- | --------------------- | ---------------------- |
| **핵심 목표** | 즉시 플레이 | 크리에이터 표현 + 판매 | 구매 / 위시리스트 |
| **레이아웃** | 깔끔, 빠른 로딩 우선 | 자유 커스터마이징 | 고밀도 정보 구조 |
| **첫 인상** | 속도 / 즉시 접근 | 테마 브랜딩 | 고품질 미디어 / 트레일러 |
| **내비게이션** | 카테고리 빠른 탐색 | 프로젝트 중심 | 깊은 카테고리 스토어 |
| **게임 목록** | 그리드 썸네일 | 커스텀 CSS 카드 | 미디어 갤러리 |

**우리 프로젝트에 적합한 모델:** CrazyGames 스타일 — 즉시 플레이 가능한 웹 게임 허브가 가장 적합하다. 별도 설치 없이 브라우저에서 바로 실행되는 구조.

### 3. 2025–2026 웹 게임 플랫폼 UI/UX 트렌드

#### 3-1. 비주얼 & 인터랙션 트렌드

| 트렌드 | 설명 | 적용 여부 |
| -------- |------ |----------- |
| **다크 모드 기본** | 게이밍 플랫폼은 다크 모드가 표준. 눈의 피로 감소 + 프리미엄 느낌 | ✅ 적용 |
| **Liquid Glass / 글래스모피즘** | 반투명 깊이감 있는 표면, 뒤 콘텐츠를 은은히 굴절 | ✅ 적용 |
| **Bento Grid 레이아웃** | 모듈형 카드 기반 레이아웃, 유기적이고 유연한 그리드 | ✅ 적용 |
| **키네틱 타이포그래피** | 스크롤/호버에 반응하는 애니메이션 헤드라인 | ⚠️ 선택적 |
| **마이크로 인터랙션** | 버튼/호버 시 촉감적 피드백 (100ms 이하 반응) | ✅ 적용 |
| **스크롤 드리븐 스토리텔링** | 스크롤에 따라 레이아웃 변형/공개 | ❌ 메인 화면에는 불필요 |

#### 3-2. UX & 기능 트렌드

| 트렌드 | 설명 | 적용 여부 |
|--------|------|-----------|
| **Sticky 내비게이션** | 상단 고정 메뉴, 주요 CTA 버튼 상시 표시 | ✅ 적용 |
| **즉시 플레이 우선** | 사용자를 게임으로 최대한 빠르게 유도 | ✅ 핵심 원칙 |
| **프로그레시브 온보딩** | 60초 이내 첫 가치 전달 (TTFV) | ✅ 적용 |
| **반응형 모바일 퍼스트** | 터치 타겟 ≥ 44x44px, 제스처 최적화 | ✅ 적용 |

### 4. 메인 화면 구성 요소 도출

연구 결과를 종합하여 메인 화면에 필요한 구성 요소를 도출했다.

#### 4-1. 레이아웃 구조 (위→아래)

```
┌─────────────────────────────────────────────────────┐
│  [Header / Navigation Bar]                          │
│  로고 · 카테고리 탭 · 검색바 · 프로필               │
├─────────────────────────────────────────────────────┤
│  [Hero Section]                                     │
│  추천 게임 슬라이더 / 배너                          │
│  큰 썸네일 + 게임명 + "지금 플레이" CTA 버튼        │
├─────────────────────────────────────────────────────┤
│  [카테고리 필터 바]                                  │
│  전체 · 퍼즐 · 아케이드 · 전략 · 멀티플레이어 ...   │
├─────────────────────────────────────────────────────┤
│  [게임 그리드]                                      │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                   │
│  │ 🎮  │ │ 🎮  │ │ 🎮  │ │ 🎮  │  ← 게임 카드     │
│  │ 이름 │ │ 이름 │ │ 이름 │ │ 이름 │  (썸네일+제목)  │
│  └─────┘ └─────┘ └─────┘ └─────┘                   │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                   │
│  │ 🎮  │ │ 🎮  │ │ 🎮  │ │ 🎮  │                   │
│  └─────┘ └─────┘ └─────┘ └─────┘                   │
├─────────────────────────────────────────────────────┤
│  [Footer]                                           │
│  저작권 · 링크 · 소셜                                │
└─────────────────────────────────────────────────────┘
```

#### 4-2. 각 구성 요소 상세

| 구성 요소 | 역할 | 핵심 사양 |
|-----------|------|-----------|
| **Header** | 전역 내비게이션 | Sticky 고정, 로고(좌), 카테고리(중), 검색+프로필(우) |
| **Hero Section** | 추천 게임 강조 | 자동 슬라이드 배너, 큰 비주얼, "Play Now" CTA |
| **카테고리 필터** | 장르별 빠른 탐색 | 수평 스크롤 가능한 필 버튼 or 탭 |
| **게임 카드 그리드** | 게임 목록 | CSS Grid 반응형, 호버 시 확대+정보 오버레이, 클릭 시 게임 실행 |
| **게임 카드** | 개별 게임 진입점 | 썸네일 이미지, 게임 이름, 카테고리 태그, 호버 애니메이션 |
| **Footer** | 부가 정보 | 저작권, 소셜 링크, 이용약관 |

### 5. 기술 스택 분석

| 기술 옵션 | 장점 | 단점 | 판단 |
|-----------|------|------|------|
| **Vanilla HTML/CSS/JS** | 단순, 의존성 없음, 빠른 초기 개발 | 규모 확장 시 관리 어려움 | ✅ 초기 단계에 적합 |
| **Vite + Vanilla** | 빌드 도구 지원, HMR, 모듈 시스템 | 초기 설정 필요 | ⚠️ 게임 수 증가 시 전환 고려 |
| **React / Next.js** | 컴포넌트 기반, 라우팅, 생태계 | 과도한 복잡성 (초기에는) | ❌ 현 단계에서는 과잉 |

**결론:** 초기에는 **Vanilla HTML + CSS + JavaScript**로 시작하여 빠르게 프로토타입을 구축하는 것이 AGENT.md A-2(단순성 우선) 원칙에 부합한다.

### 6. 디자인 시스템 방향

#### 6-1. 색상 팔레트 (다크 테마)

| 용도 | 색상 | 코드 |
|------|------|------|
| 배경 (최하위) | 매우 어두운 남색 | `#0a0e1a` |
| 배경 (카드/표면) | 진한 남색 | `#111827` |
| 표면 (Glassmorphism) | 반투명 백색 | `rgba(255,255,255,0.05)` |
| 주요 액센트 | 밝은 시안 / 네온 블루 | `#00d4ff` |
| 보조 액센트 | 일렉트릭 퍼플 | `#7c3aed` |
| 텍스트 (기본) | 연한 백색 | `#e2e8f0` |
| 텍스트 (보조) | 중간 회색 | `#94a3b8` |

#### 6-2. 타이포그래피

- **Primary Font:** `Inter` (Google Fonts) — 깔끔하고 현대적인 산세리프
- **Headings:** 700 (Bold), 게임 제목은 600 (Semi-Bold)
- **Body:** 400 (Regular)

#### 6-3. 핵심 애니메이션

| 요소 | 애니메이션 | 속도 |
|------|-----------|------|
| 게임 카드 호버 | scale(1.05) + box-shadow 증가 | 0.3s ease |
| 내비게이션 링크 | 하단 보더 슬라이드 인 | 0.2s ease |
| 히어로 배너 | 좌→우 슬라이드 전환 | 5s 간격 자동 |
| 카테고리 버튼 | 배경색 페이드 + 테두리 글로우 | 0.2s ease |
| 페이지 진입 | 카드 순차 fade-in + slide-up | 0.4s stagger |

### 7. 게임 데이터 구조 제안

게임 목록을 데이터 드리븐으로 관리하기 위한 JSON 구조:

```json
{
  "games": [
    {
      "id": "game-001",
      "title": "게임 이름",
      "category": "puzzle",
      "thumbnail": "assets/images/game-001.png",
      "description": "짧은 설명",
      "featured": true,
      "path": "games/game-001/index.html"
    }
  ]
}
```

→ 게임 추가/삭제 시 HTML 수정 없이 JSON만 업데이트하면 되는 구조.
