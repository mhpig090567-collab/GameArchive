## 2026-05-26 16:48 KST — 웹 기반 멀티 게임 플랫폼 메인 화면 구현 계획

### When (일시)

2026-05-26

### Why (목적)

하나의 웹 페이지에서 여러 게임을 탐색하고 즉시 플레이할 수 있는 멀티 게임 플랫폼의 **메인 화면(랜딩 페이지)**을 구현한다. RESEARCH.md의 분석 결과에 따라, CrazyGames 스타일의 즉시 플레이형 게임 허브를 목표로 한다.

### What (구현 대상)

메인 화면 1페이지 완성. 아래 6개 구성 요소를 포함:

| # | 구성 요소 | 핵심 기능 |
|---|-----------|-----------|
| 1 | **Header (내비게이션 바)** | 로고, 카테고리 탭, 검색 바, 프로필 영역. Sticky 고정. |
| 2 | **Hero Section (히어로 배너)** | 추천 게임 슬라이더. 자동 전환(5초). 큰 비주얼 + "Play Now" CTA. |
| 3 | **카테고리 필터 바** | 장르별 필터 버튼 (전체, 퍼즐, 아케이드, 전략, 멀티플레이어 등). 클릭 시 게임 그리드 필터링. |
| 4 | **게임 카드 그리드** | CSS Grid 반응형 레이아웃. JSON 데이터 기반 동적 렌더링. |
| 5 | **게임 카드** | 썸네일 + 게임명 + 카테고리 태그. 호버 시 확대 + 글로우 + 정보 오버레이. |
| 6 | **Footer** | 저작권, 소셜 링크, 부가 정보. |

### How (기술 접근)

#### 기술 스택

| 레이어 | 기술 | 이유 |
|--------|------|------|
| 구조 | HTML5 (시맨틱) | `<header>`, `<main>`, `<section>`, `<footer>` 활용 |
| 스타일 | Vanilla CSS | CSS Custom Properties(변수), Grid, Flexbox, Glassmorphism |
| 로직 | Vanilla JavaScript (ES6+) | 모듈 없이 단일 스크립트. JSON 기반 동적 렌더링 |
| 폰트 | Google Fonts — Inter | CDN 로드 |
| 데이터 | `data/games.json` | 게임 메타데이터 파일. HTML 수정 없이 게임 추가/삭제 가능 |

#### 디자인 시스템

| 요소 | 사양 |
|------|------|
| **테마** | 다크 모드 기본 |
| **배경** | `#0a0e1a` (최하위), `#111827` (카드/표면) |
| **Glassmorphism** | `rgba(255,255,255,0.05)` 배경 + `backdrop-filter: blur(12px)` |
| **주 액센트** | `#00d4ff` (네온 시안) |
| **보조 액센트** | `#7c3aed` (일렉트릭 퍼플) |
| **그래디언트** | 시안→퍼플 방향 (`#00d4ff` → `#7c3aed`) |
| **텍스트** | 기본 `#e2e8f0`, 보조 `#94a3b8` |
| **폰트** | Inter 400/600/700 |
| **카드 호버** | `scale(1.05)` + `box-shadow` 글로우 + 0.3s ease |
| **페이지 진입** | 카드 순차 fade-in + slide-up (stagger 0.1s) |
| **반응형** | 모바일: 1열 → 태블릿: 2열 → 데스크톱: 4열 |

#### 파일 구조

```

├── css/

│   └── main.js             ← 메인 로직 (렌더링, 필터, 슬라이더)

├── assets/

└── Agent/                  ← 기존 문서 디렉토리 (변경 없음)
    ├── AI/
    ├── Project/
    └── User/
```

### In What Order (구현 순서)

아래 순서대로 단계적으로 구현한다. 각 단계마다 검증 기준을 명시.

```
Step 1 → Step 2 → Step 3 → Step 4 → Step 5 → Step 6
```

#### Step 1. 프로젝트 기반 설정

- **작업:** Git 브랜치 생성 (`feature/main-page`), 디렉토리 구조 생성, `games.json` 샘플 데이터 작성
- **산출물:** `data/games.json`, 빈 `css/style.css`, 빈 `js/main.js`, 빈 `index.html`
- **검증:** 모든 파일이 생성되고 Git 상태 확인

#### Step 2. CSS 디자인 시스템 구축

- **작업:** CSS Custom Properties 정의, 리셋, 타이포그래피, 유틸리티, 반응형 브레이크포인트
- **산출물:** `css/style.css` (디자인 토큰 + 기본 스타일)
- **검증:** 브라우저에서 빈 페이지 로드 시 다크 배경 + Inter 폰트 적용 확인

#### Step 3. HTML 구조 작성

- **작업:** `index.html`에 시맨틱 HTML 구조 (header, hero, filter, grid, footer)
- **산출물:** `index.html` 완성
- **검증:** 정적 HTML이 올바르게 표시되는지 확인

#### Step 4. CSS 컴포넌트 스타일링

- **작업:** Header, Hero, 카테고리 필터, 게임 카드 그리드, Footer 스타일링. 호버 애니메이션, Glassmorphism, 반응형 레이아웃 적용
- **산출물:** `css/style.css` (컴포넌트 스타일 추가)
- **검증:** 데스크톱/모바일 뷰에서 모든 요소가 디자인대로 표시, 호버 애니메이션 동작

#### Step 5. JavaScript 동적 기능 구현

- **작업:**
  - `games.json` 로드 → 게임 카드 동적 렌더링
  - 카테고리 필터 클릭 시 게임 목록 필터링
  - 히어로 배너 자동 슬라이드
  - 페이지 진입 시 카드 순차 애니메이션
- **산출물:** `js/main.js`
- **검증:** 카테고리 필터 동작, 슬라이더 자동 전환, 카드 클릭 동작 확인

#### Step 6. 게임 썸네일 이미지 생성 및 최종 통합

- **작업:** 게임 썸네일 이미지 생성(generate_image), `games.json`에 경로 연결, 전체 통합 테스트
- **산출물:** `assets/images/` 내 썸네일 파일들
- **검증:** 모든 이미지 로드, 전체 플로우 (탐색→필터→카드 호버→클릭) 정상 동작
